# v0.3.0-alpha

This is an architecture and runtime preview release for the phone-native Mobile Agent.

## Highlights

- Major Android runtime refactor: native tools, task loop, recovery, memory, MCP, SSH, web, workspace, and UI code are split into focused modules.
- Added stronger phone observation primitives, including structured accessibility snapshots and intent/file action tools.
- Added formal progressive loading for native tools, plugins, skills, and MCP tools, reducing prompt bloat and preserving stable prompt prefixes.
- Added Skill registry and Skill execution tools for reusable phone and PC workflows.
- Expanded phone-to-PC bridge support with SSH diagnostics, command execution, SFTP file transfer, MCP recovery, and terminal fallback.
- Improved multi-MCP support for Desktop Control style tool aggregation.
- Added stop/interruption plumbing and clearer failure reporting for terminal, MCP, and SSH paths.
- Fixed Termux HTTP backend recovery by removing self-killing inline `pkill` behavior from Android RUN_COMMAND recovery.
- Added `scripts/enable-termux-run-command.sh` for the Termux one-time external command setting.
- Rewrote README as clean Chinese-first public documentation.

## Alpha Notes

- This is still an alpha release. It is intended for early testing and architecture feedback.
- Desktop MCP control requires separately running desktop MCP services and configuring the phone-side endpoint/token.
- SSH/Tailscale PC control depends on the user's own network, SSH server, key setup, and firewall settings.
- Termux recovery requires Termux plus `allow-external-apps=true`; run `sh scripts/enable-termux-run-command.sh` once in Termux.
- Some advanced workflows are intentionally exposed as tools/skills first, before being polished into a consumer UI.
