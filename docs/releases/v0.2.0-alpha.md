# v0.2.0-alpha

This is a preview release for phone-led PC control.

## Highlights

- Native SSH bridge from the Android app to a Windows PC over LAN or Tailscale.
- PowerShell command execution and SFTP file transfer through the phone Agent.
- SSH local forwarding for Windows MCP, so the phone can use desktop tools through `127.0.0.1:18000/mcp`.
- `pc_bridge_recover` to select a reachable SSH host, connect, start the tunnel, sync MCP auth, diagnose the remote MCP endpoint, and verify recovery.
- Multi-MCP server registry primitives: `mcp_servers`, per-server `mcp_status`, `mcp_tools`, `mcp_tool_info`, and `mcp_call`.
- Progressive loading for native tools, plugins, skills, and MCP tools.
- Task interruption, context compaction, cache display improvements, and simplified Chinese output normalization.

## Verified In This Build

- Android debug build succeeds.
- Python test suite: 45 passed.
- Phone-to-PC SSH over Tailscale works.
- Windows MCP is reachable through the phone tunnel.
- MCP tool discovery returns 19 desktop tools including `App`, `PowerShell`, `FileSystem`, `Snapshot`, `Screenshot`, `Click`, `Type`, and `Scroll`.

## Alpha Notes

- This release is intended for early testing, not a stable public workflow.
- Windows MCP still requires an external MCP service and auth token.
- Termux terminal SSH fallback is implemented, but requires Termux backend availability plus OpenSSH/key setup.
- Multi-MCP routing is available at the tool layer; higher-level automatic server selection is still early.
